Author

Wilder Clavijo y María José Duarte H

Se realizaron correcciones en la visualización de las etiquetas en el mapa y se realizó de nuevo la exportación, de forma que las etiquetas fueran visibles a partir de cierta escala.

Cuestionario de Evaluación y Análisis

Debido a algunos errores al utilizar QGIS Server fue necesario responder las preguntas recurriendo a las fuentes referenciadas al final del documento

Pregunta 1: Explique las diferencias arquitectónicas entre qgis2web (renderizado cliente) y QGIS Server (renderizado servidor). ¿En qué casos es preferible cada enfoque en términos de rendimiento, mantenibilidad e interactividad?

qgis2web exporta los datos vectoriales a formatos planos como GeoJSON, todo el procesamiento gráfico, los estilos y el cálculo de los mapas se ejecuta directamente en el navegador web de la computadora del usuario, no requiere un servidor backned para funcionar. Por otro lado, QGIS Server funciona bajo un esquema en el que un servidor web procesa las peticiones geográficas, cuando un cliente solicita ver el mapa el servidor lee las bases de datos originales y el envía al usuario el resultado renderizdo como una imagen estática.

Para seleccionar cual de las dos opciones utilizar en términos de rendimiento, qgis2web es ideal para mapas con pocos elementos vectoriales ya que depende del hardware del cliente, si la cantidad de datos es muy alta el navegador puede fallar, por su parte, QGIS Server optimiza el rendimiento mediante el uso de protocolos como WMTS que entrega mapas estructurados lo que reduce la carga del procasamiento del servidor. Este también ofrece una mantenibilidad superior si se trabaja a nivel empresarial ya que lee un único archivo del proyecto y la geodatabase mientras que con qgis2web cualquier actualización o cambio que se realice hace necesario realizar el proceso de exportación nuevamente.

En interactividad, qgis2web da información con navegación fluida, consulta automática de pop-ups si se configura de forma adecuada, herramientas de medición y búsqueda. En QGIS Server este parámetro dependen del uso de protocolos específicos como WFS.

Pregunta 2: ¿Qué ventajas ofrece la publicación de servicios OGC estándar (WMS, WFS) frente a aplicaciones web propietarias? Considere interoperabilidad, escalabilidad y portabilidad de datos.

La adopción de los estándares del Open Geospatial Consortium (OGC) garantiza la interoperabilidad entre sistemas de información geográfica, permitiendo que cualquier cliente compatible, ya sea de software libre o comercial, acceda y consuma datos espaciales a través de la red sin necesidad de descargar los archivos originales. Esto facilita el intercambio de información geográfica bajo reglas comunes y asegura una integración entre diferentes plataformas y herramientas.

Además, los estándares OGC favorecen la escalabilidad y la portabilidad de los datos, tecnologías como WMTS optimizan el rendimiento al servir capas cartográficas mediante teselas almacenadas en caché, reduciendo la carga de procesamiento y permitiendo atender múltiples usuarios simultáneamente sin afectar la calidad del servicio. Por su parte, el estándar WFS permite transmitir geometrías vectoriales y atributos en formatos abiertos como GML o GeoJSON, facilitando la descarga, consulta, edición y exportación de la información hacia bases de datos locales, sin depender de formatos propietarios o de un proveedor específico.

Pregunta 3: En el contexto de los datos geoespaciales con MAGNA-SIRGAS (EPSG:9377), explique los desafíos de transformación de proyecciones cuando los servicios web requieren Web Mercator (EPSG:3857). ¿Cómo QGIS maneja estas conversiones automáticamente?

La transformación entre el sistema oficial de Colombia (EPSG:9377), y el sistema Pseudo-Mercator (EPSG:3857) utilizado por la mayoría de los visores web es un desafio para la publicación de mapas virtuales e infromación cartográfica digital, ya que este proceso requiere cálculos continuos que incrementan el costo computacional y pueden generar ligeros desplazamientos espaciales. Para resolverlo, QGIS Server permite definir explícitamente los sistemas de referencia compatibles, de modo que el software realiza automáticamente las transformaciones necesarias y proyecta las capas de forma transparente, garantizando su correcta visualización y alineación con los mapas base globales en aplicaciones web.

Pregunta 4: Describa el flujo completo de datos desde un GeoPackage local hasta su consumo en una aplicación web. ¿En qué puntos del flujo se puede perder información o precisión?

  1. Los datos geográficos se extraen desde tablas vectoriales que se encuentran dentro de un archivo geopackage local.

  2. Se víncula el sistema de coordenadas oficial y se definen simbologías y clasificaciones temáticas, también se editan las plantillas HTML para visualizar los pop-ups.

  3. El diseño se almacena en un archivo de proyecto que va a ser la base del servidor.

  4. Para el caso de sevidores, los archivos se mueven a la ruta web y se vincula el motor.

  5. El navegador web del usuario procesa la aplicación interactiva que puede ser generada por plugings o complementos como qgis2web o mediante la URL de petición de OGC.

Pueden producirse algunas pérdidas de información durante la reproyección al vuelo desde el sistema EPSG:9377 hacia sistemas globales, esto puede introducir pequeños errores de redondeo que reducen ligeramente la precisión espacial. De igual forma, al utilizar servicios WMS o WMTS, los datos vectoriales se transforman en imágenes ráster, lo que limita el acceso directo a los atributos y valores originales de la geodatabase. Por último, para mejorar el rendimiento en los navegadores web, suele ser necesario simplificar geometrías o agrupar elementos complejos, sacrificando el detalle cartográfico para tener una mayor velocidad de carga y visualización.
